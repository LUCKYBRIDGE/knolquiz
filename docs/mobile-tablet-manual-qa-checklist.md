# 모바일/태블릿 실사용 QA 체크리스트

범위: `knolquiz-runtime` 운영 화면(런처/점프맵/퀴즈/기록/학급관리)

## 1) 목적

- 자동 점검(`scripts/mobile-tablet-qa-check.mjs`)으로 잡지 못하는 UI/터치 동작 문제를 실기기에서 확인한다.
- 문제 발생 시 재현 정보를 같은 형식으로 남겨 수정/회귀 점검 시간을 줄인다.

## 2) 사전 준비

1. 로컬 서버 실행
   - `cd /Users/baekjiyun/Desktop/WAN/knolquiz-runtime`
   - `node scripts/jumpmap-local-serve.mjs`
2. 자동 정적 점검 실행
   - `node scripts/mobile-tablet-qa-check.mjs`
3. 분리 검증 실행(권장)
   - `cd /Users/baekjiyun/Desktop/WAN/math-net-master-quiz`
   - `node scripts/jumpmap-verify-split.mjs --skip-smoke`

## 3) 실기기 매트릭스

아래 조합을 최소 1회씩 점검한다.

1. 스마트폰(세로)
   - 1인, 2인
2. 스마트폰(가로)
   - 1인, 2인
3. 태블릿(세로)
   - 1인, 3인
4. 태블릿(가로)
   - 1인, 4인, 6인

## 4) 공통 체크 항목

1. 런처 진입 후 게임 시작까지 콘솔 에러 없음
2. 플레이어 수(1~6) 선택에 따라 이름/학생번호 입력 UI 잘림 없음
3. 학생번호 태그 입력 후 플레이 시작 시 결과화면 링크 컨텍스트 유지
4. 화면 회전(세로↔가로) 후 버튼/텍스트 레이아웃 복구
5. 브라우저 주소 새로고침 후 페이지 치명적 깨짐 없음

## 5) 점프맵 체크 항목

1. 방향키/점프/퀴즈 버튼이 겹치지 않음
2. 방향키 좌/우 버튼 간격이 충분함(오입력 없음)
3. 조작 편집 모드에서 위치/크기 조절 가능
4. 1인~6인에서 조작 UI 잘림/겹침 없음
5. 결과 오버레이에서
   - `결과 CSV` 다운로드 동작
   - `지난 기록 보기`, `학급관리 / 명예의 전당` 링크 동작
   - 학생번호/기간 컨텍스트 유지 동작

## 6) 퀴즈 체크 항목

1. 문제 카드/선택지 터치 입력 정상
2. 스크롤/드래그 필요한 화면에서 입력 충돌 없음
3. 결과 화면에서
   - JSON 복사/다운로드 동작
   - `CSV 다운로드` 동작
   - `지난 기록 보기`, `학급관리 / 명예의 전당` 링크 동작
   - 학생번호/기간 컨텍스트 유지 동작

## 7) 기록/학급관리 체크 항목

1. `지난 기록` 필터(학생/기간) 변경 시 데이터 즉시 반영
2. `학생 상세 기록` 필터(학생/기간) 변경 시 데이터 즉시 반영
3. `학급관리` 명예의 전당 기간 필터 변경 시
   - 목록 반영
   - 상세 링크 반영
   - CSV 파일명/내용 반영
4. 기록 CSV 내보내기(지난 기록/학생 상세) 동작

  1)
  2)
  3)

- 기대 결과:

## 9) 종료 기준

아래 3개가 모두 충족되면 모바일/태블릿 QA를 통과로 본다.

1. 자동 점검 `pass=all, fail=0`
2. 실기기 매트릭스 전 항목 점검 완료
3. 치명 이슈(진행 불가/입력 불가/링크 오동작) 0건
