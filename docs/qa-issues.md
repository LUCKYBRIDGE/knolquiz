# QA 이슈 운영 로그

작성일: 2026-02-28  
범위: 모바일/태블릿 실사용 QA 이슈 기록

## 1) 사용 규칙

1. 이슈 1건당 표 1행으로 기록한다.
2. 재현 가능한 최소 단계를 반드시 적는다.
3. 수정 커밋이 나오면 `상태`, `해결커밋`, `메모`를 즉시 갱신한다.
4. 새 QA 사이클 시작 전 `OPEN`/`IN_PROGRESS` 이슈를 먼저 재검증한다.

## 2) 상태 값

- `OPEN`: 미해결
- `IN_PROGRESS`: 수정 중
- `FIXED_PENDING_QA`: 수정 완료, 재검증 대기
- `CLOSED`: 재검증 완료
- `WONT_FIX`: 의도 동작/보류

## 3) 우선순위 값

- `P0`: 진행 불가/치명 오류
- `P1`: 주요 기능 저해
- `P2`: 사용성 저하
- `P3`: 경미한 개선

## 4) 이슈 로그

| ID | 상태 | 우선순위 | 발견일 | 화면/기능 | 환경(기기/브라우저) | 재현 단계(요약) | 실제 결과 | 기대 결과 | 해결커밋 | 메모 |
|---|---|---|---|---|---|---|---|---|---|---|
| QA-001 | OPEN | P2 | 2026-02-28 | 예: 점프맵 결과 오버레이 | 예: iPad Safari | 예: 1) 4인 시작 2) 꼭대기 도달 | 예: 버튼 겹침 | 예: 버튼 분리 배치 |  |  |

## 5) 상세 재현 템플릿

```text
[이슈 제목]
- ID:
- 날짜/시간:
- 상태/우선순위:
- 빌드/커밋:
- 환경:
  - 기기:
  - OS/버전:
  - 브라우저/버전:
  - 화면 방향(세로/가로):
- 진입 URL:
- 사전 상태:
  - 플레이어 수:
  - 학생번호:
  - 기간필터:
- 재현 단계:
  1)
  2)
  3)
- 실제 결과:
- 기대 결과:
- 재현 빈도: (항상/간헐)
- 첨부:
  - 스크린샷/영상:
  - 콘솔 로그:
- 해결 커밋:
- 회귀 테스트 결과:
```
